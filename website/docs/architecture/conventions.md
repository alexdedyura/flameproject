# Соглашения разработки

Правила, которых придерживается кодовая база. Соблюдайте их, чтобы новый код органично вписывался.

## Callbacks

Расширяйте callback'и через `YSI\y_hooks` там, где так делает соседний код:

```c
hook OnPlayerConnect(playerid)
{
    // ...
    return 1;
}
```

Либо диспетчеризуйте в per-module функции, как это уже сделано в `flame.pwn`
(`Vehicles_OnPlayerStateChange(...)` и т.п.). Следуйте паттерну окружающего модуля.

## Команды

Команды добавляются через `izcmd`:

```c
CMD:name(playerid, params[])
{
    // разбор аргументов через sscanf2
    return 1;
}
```

Аргументы парсятся через `sscanf2`. Группируйте новые команды по темам в `sources/commands/`
(см. [Команды](../systems/commands)).

## Работа с MySQL

Используйте существующие хелперы из `sources/core/mysql.inc`:

- `UpdateBaseInt/Str/Float` — точечные обновления полей;
- `mysql_format` + `mysql_tquery` с callback-паттернами для запросов.

Имя базы — `flame`. Схема/дамп — `database/flame.sql`.

## Модульность

- **Новый подмодуль** подключайте через ближайший агрегирующий `.inc`, а не напрямую из `flame.pwn`.
- `forward` ставьте перед `public` для timer/MySQL-callback функций, если этого требует компилятор
  и так принято в соседнем коде.

## Совместимость с open.mp (`SAMP_COMPAT`)

Держите compat-шим довольным:

- используйте `true`/`false` для булевых значений;
- итерируйтесь через `foreach (new i : Vehicle)`-стиль;
- выносите литералы `mysql_format` во временные переменные там, где этого требуют compat-заголовки.

## Что не трогать без явной необходимости

- Бандл `qawno/` (инклюды + `*.exe`), `omp-server.exe`, бинарники в `components/` и `plugins/`;
- `gamemodes/flame.amx` — только артефакт сборки, никогда не редактируется вручную;
- массовые изменения `config.json`, `database/flame.sql`, runtime-данных `scriptfiles/`;
- несвязанные изменения в git-дереве — проверяйте `git status --short` перед правками.

:::danger Кодировка
Никогда не сохраняйте `.inc`/`.pwn` с кириллицей в UTF-8 — это необратимо ломает текст.
Обязательно прочитайте [Кодировка cp1251](encoding) перед правкой файлов с русским текстом.
:::
