# Инвентарь

Система предметов игрока. Логика — в `sources/player/inv/`, интерфейс — в `textdraws/inventory.inc`.

## Структура модуля

| Файл | Назначение |
| --- | --- |
| `inv/items.inc` | Определения предметов (ID, свойства). |
| `inv/use.inc` | Использование предмета. |
| `inv/drop.inc` | Выброс предмета. |
| `inv/show.inc` | Отображение инвентаря (TextDraw-UI). |
| `inv/inv.inc` | Агрегатор и общая логика инвентаря. |

## Как хранится

Предметы игрока сохраняются в таблицу `inventory` (см. [База данных](../database)).
Слоты инвентаря доступны как `PlayerInventory[playerid][slot][...]`, где помимо самого предмета
хранится доп. параметр (`invInfo`) — например, для телефона в нём лежит выбор обоев.

## Интерфейс

Инвентарь отрисовывается через TextDraw-UI (`textdraws/inventory.inc`). Игрок видит слоты с предметами,
может использовать или выбросить предмет. Подробнее об UI — в разделе [Интерфейс](interface).

## Пример: предмет «Телефон»

Телефон — это предмет инвентаря с фиксированным ID:

| Константа | Значение |
| --- | --- |
| `INV_ITEM_PHONE` | `6` |
| `INV_PHONE_PARAM` | `1` |

Использование телефона открывает подсистему [Смартфона](phone). Тестовая выдача — командой `/givephone`.
