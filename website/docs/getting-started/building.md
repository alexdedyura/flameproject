# Сборка гейммода

Весь гейммод компилируется в единственный `flame.amx` из одной точки входа — `sources/flame.pwn`.

## Быстрая сборка

Под Windows достаточно запустить:

```bat
compile.bat
```

Скрипт вызывает компилятор Pawn `qawno\pawncc.exe` для `sources\flame.pwn` и переносит собранный
`flame.amx` в каталог `gamemodes\`.

## Что делает компилятор

Флаги сборки (те же, что использует CI):

```bat
qawno\pawncc.exe "-;+" "-(+" "-ogamemodes\flame.amx" "sources\flame.pwn"
```

- `-;+` — требовать `;` в конце выражений;
- `-(+` — требовать скобки `()`;
- `-o…` — путь для результата (`gamemodes\flame.amx`);
- include-путь по умолчанию — `qawno\include` (относительно `pawncc.exe`), поэтому пути вида
  `<..\..\sources\...>` из `flame.pwn` резолвятся из корня репозитория.

## Что считается успехом

Отдельного линтинга или тестов нет. **Сборка прошла**, если компилятор создал `gamemodes/flame.amx`
без новых ошибок и предупреждений. После правок Pawn-кода всегда пересобирайте гейммод и убеждайтесь,
что он по-прежнему компилируется.

:::danger Никогда не редактируйте `flame.amx` вручную
`gamemodes/flame.amx` — это артефакт сборки. Правьте только исходники в `sources/` и пересобирайте.
:::

## Автоматическая сборка (CI)

Сборка автоматизирована через GitHub Actions (`.github/workflows/build.yml`): на каждый push в `main`
и pull request гейммод компилируется на `windows-latest`, а готовый сервер пакуется в `.zip`.
Подробнее — в разделе [Разработка → CI/CD](../contributing#cicd-github-actions).
